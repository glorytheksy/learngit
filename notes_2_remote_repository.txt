1. 添加远程库
在github上创建远程仓库之后，需要把本地内容推送到远端，我们需要两个指令。 

git remote add origin 仓库地址
git push -u origin master

origin 是远程仓库的名字，这是git的默认叫法
特别地，git push 是本地推送给远程的命令，实际上是把 “当前分支” 推送给远程仓库
-u 的意思是表示让本地仓库与远程仓库origin产生默认关联。什么叫默认关联？我们知道，本地仓库可能与多个远程存在追踪关系，那么这时如果不建立默认关系，直接使用命令 git pull ，本地则不知道该从哪个主机拉代码。 但如果设置了默认关系，本地就从默认远程仓库拉代码。默认关系设置一次就够了，以后不需要再次加 -u 参数。

另外，推送有两种模式：simple模式和matching模式。simple模式下，只把当前分支推送给远程仓库；matching模式下会把本地所有分支推送给远程仓库。当前版本默认为simple模式
